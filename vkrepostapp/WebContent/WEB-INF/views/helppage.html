Получение access token.

Актуально на момент 23.09.16

1. Зарегестрированный с телефоном аккаунт.

2. Созданный standalone application.

3. В браузере пройти по ссылке следующего вида(при этом быть залогиненым):

https://oauth.vk.com/authorize?client_id=[ID приложения]&response_type=token&v=5.53 &scope=offline,groups,wall,photos&redirect_uri=https://api.vk.com/blank.html

scope установленный на offline озачает что будет выдан практически бессрочный token.

4. Ответом будет выдан access token.

Подробнее

Поля

access token

Здесь должен быть access token полученный от vk.com (см. выше)

id группы для постинга

id группы куда будет происходить перепостинг, держатель access token-а должен иметь нужные права.

id владельца группы

см. документацию vk.com чтобы узнать как извлекать id групп.

Задержка между постингом в миллисекундах

1 минута это 60000млс. Для правильной работы не стоит ставить слишком малые значения.

Количество постов за итерацию с каждой группы

Количество последних постов. Из-за ограничений со стороны vk.com не имеет смысла ставить слишком большие значения. Если включен фильтр или данная группа не содержит постов с картинками может ничего не произойти. Следи за логами.

Задержка между итерациями

Значение желательно должно быть больше в несколько раз чем задержка между постингом. Указывать в миллисекундах.

Не использовать фильтр

Когда этот праметр активен могут пройти посты содержащие не только единственную картинку.
